# Deploy Plus One on Render

This deployment keeps `DEEPSEEK_API_KEY` on the server. Visitors use the app through the Render URL and never receive the API key in the browser.

## 1. Push this branch to GitHub

Commit the deployment files, then push the branch:

```bash
git add .
git commit -m "Add Render deployment config"
git push -u origin deepseek-api
```

Open a pull request and merge it into `main`, or deploy this branch directly from Render.

## 2. Create the Render service

Recommended path:

1. Open Render Dashboard.
2. Choose **Blueprints**.
3. Connect this GitHub repository.
4. Select `render.yaml`.
5. When Render asks for `DEEPSEEK_API_KEY`, paste the key into Render's environment variable UI.

The API key should not be committed to GitHub.

## 3. Confirm environment variables

Render should have:

```text
DEEPSEEK_API_KEY=your_real_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
PLUSONE_LLM_MODEL=deepseek-v4-flash
DJANGO_DEBUG=False
SECRET_KEY=<generated by Render>
DATABASE_URL=<provided by Render PostgreSQL>
```

`RENDER_EXTERNAL_HOSTNAME` is set automatically by Render and is used by Django for `ALLOWED_HOSTS`.

## 4. Share the app

After the first deploy completes, Render provides a public `.onrender.com` URL. Share that URL with other people. They do not need Python, Django, or your DeepSeek key on their computers.

## 5. Rotate the key later

Use a dedicated DeepSeek key for this deployment. If the key is exposed or the review period ends, delete or rotate it in the DeepSeek dashboard and update the Render environment variable.
