# Deploy Plus One on Render

This path publishes one hosted Plus One instance with PostgreSQL, server-side DeepSeek access, collected static files, and HTTPS through Render. Visitors use the public Render URL and never receive `DEEPSEEK_API_KEY` in the browser.

## 1. Preflight locally

Run these from the Django project root:

```bash
python manage.py check
python manage.py test
python manage.py collectstatic --no-input
```

Make sure real secrets are not committed:

```bash
git status --short
```

The real `.env` file is ignored by git. `.env.example` must contain placeholders only.

## 2. Push to GitHub

Commit the deployment-ready files and push the branch:

```bash
git add .
git commit -m "Prepare Plus One for production deploy"
git push
```

Deploy from the branch you want Render to track, usually `main`.

## 3. Create the Render service

Recommended path:

1. Open the Render Dashboard.
2. Choose **Blueprints**.
3. Connect the GitHub repository.
4. Select `render.yaml`.
5. When Render asks for `DEEPSEEK_API_KEY`, paste the real key into Render's environment variable UI.

`render.yaml` creates:

- a Python web service named `plusone`,
- a PostgreSQL database named `plusone-db`,
- free instance plans for the first public test,
- a generated `SECRET_KEY`,
- `DJANGO_DEBUG=False`,
- `DATABASE_URL` wired from the Render PostgreSQL database,
- HTTPS redirect and short HSTS defaults in Django production mode,
- `build.sh` as the build command,
- Gunicorn + Uvicorn as the start command.

## 4. Confirm environment variables

Render should have:

```text
DEEPSEEK_API_KEY=your_real_key
DEEPSEEK_BASE_URL=https://api.deepseek.com
PLUSONE_LLM_MODEL=deepseek-v4-flash
DJANGO_DEBUG=False
SECRET_KEY=<generated by Render>
DATABASE_URL=<provided by Render PostgreSQL>
SECURE_SSL_REDIRECT=True
SECURE_HSTS_SECONDS=3600
```

`RENDER_EXTERNAL_HOSTNAME` is set automatically by Render and is used by Django for `ALLOWED_HOSTS` and CSRF trusted origins.

If you add a custom domain later, add:

```text
ALLOWED_HOSTS=your-domain.com
CSRF_TRUSTED_ORIGINS=https://your-domain.com
```

Only enable these after you are certain every subdomain under your domain is HTTPS-only:

```text
SECURE_HSTS_INCLUDE_SUBDOMAINS=True
SECURE_HSTS_PRELOAD=True
```

## 5. Verify the live app

After the first deploy completes:

1. Open the `.onrender.com` URL.
2. Create a Plus One card from casual text.
3. Open another browser or private window to simulate a second anonymous user.
4. Swipe interested, open chat, send a safe message, and agree to meet.
5. Try an unsafe post or chat message and confirm it is blocked.
6. Visit `/dashboard/` and confirm live cards, open chats, handoffs, and history render cleanly.

## 6. Production data note

`build.sh` runs migrations only. It does not run `seed_demo`, so production will not be filled with demo users and demo posts.

If you intentionally want a demo environment, run this manually in a non-production database:

```bash
python manage.py seed_demo
```

The free Render Postgres plan is suitable for launch testing, but it has platform limits and may expire according to Render's current free database policy. Upgrade the database before treating user data as durable.

## 7. Rotate the key later

Use a dedicated DeepSeek key for this deployment. If the key is exposed or the review period ends, delete or rotate it in the DeepSeek dashboard and update the Render environment variable.
